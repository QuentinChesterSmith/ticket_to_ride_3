This is my basic implementation of MCTS (Monte Carlo Tree Search) in the board game Ticket To Ride. Ticket To Ride is a board game about building train networks, claiming routes, and completing destination tickets. I have attempted to build a model for this game in the past using methods such as; neural networks, NEAT, vanilla PPO, and I finally landed on MCTS as the best model for this game. This is a very weak model and almost all human players would be able to beat it, however it is a basic outline for how a real model would look. Within the data.txt file you can see the simplistic experiments that I ran, and the data I gathered. In reality this data is of very little use as there are only 100 games played per experiment and very few rollouts. I built this project as a way to learn about different RL (Reinforcement Learning) techniques and specifically MCTS, and building a model for this game has been a passion project of mine for the past year or so. If I were to return to this project later some improvements I would make would be improving the basic heuristics as they are very minimal and merely used as demonstration. I would also work on optimizing the rollouts for speed, specifically cutting down on the use deep copying the game state as I often as I do, as that is a huge bottlenck in speed and efficiency. Finally if I truly wanted to optimize for speed I would rewrite most of the code in a more efficient programming language like C.
